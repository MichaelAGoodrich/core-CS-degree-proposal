# Impact on Existing Courses

This README consolidates the committee's current notes on how the proposed degree structure may affect existing courses.

## CS 252 — Introduction to Computational Theory

Considerations for better connecting theory to practice:

- Shift some reducibility-proof emphasis to topics that help students connect theory to modern computing practice.
- Introduce parallel computability and quantum computability as examples of how theoretical models shape real systems.
- Include lambda calculus to support functional programming concepts used elsewhere in the curriculum.

## CS 260 — Web Programming

Considerations for making room for ACM-recommended HCI elements:

- Reserve approximately one credit hour for core HCI topics recommended by the ACM curriculum.
- Adjust existing web programming content to create that space without reducing essential skills.
- Emphasize usability, accessibility, and user-centered design in the web context.

## CS 224 / CS 324 — Systems Sequence

The committee is aware that there is department discussion about the future of these classes. We have not been informed of the reasons for that discussion, so we have not been able to address those issues directly.

Two brief brainstorm ideas that may be useful for future discussion:

- Make space in CS 324 for GPU-based programming to reflect modern systems needs, including AI workloads.
- Consider whether key ideas from other systems courses are important enough to include in the two required systems classes, which would require informed choices about what to drop.

Any topic changes should align with the guiding principle that CS is the science of abstraction, with abstractions relevant to both theoretical maturity and programming practice.
