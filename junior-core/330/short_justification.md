# CS 330 Draft Justification

**Status note:** CS 330 already exists, and **no changes are required at this time**. This document explains why it belongs in the junior core.

## Purpose in the Junior Core

CS 330 anchors the **programming languages and abstraction** pillar in the junior core. It gives students a principled understanding of how languages express computation, how paradigms shape design, and how language features influence software reliability and maintainability.

## Justification Based on Current Course Overview

The existing CS 330 description emphasizes core language concepts and paradigms that every computing professional should understand:

- **Programming language principles** that generalize across languages and tools
- **Process and data abstraction** and their role in reliable system design
- **Encapsulation and inheritance** as foundational mechanisms for modularity
- **Functional and logic programming** as contrasting paradigms
- **Scanners and parsers** that connect language theory to practical implementation

These topics are central to the program’s emphasis on enduring abstractions and provide a conceptual bridge between algorithms, systems, and software engineering.

## Why It Belongs in the Junior Core

By placing CS 330 in the junior core, all students gain a shared conceptual vocabulary for reasoning about programming languages and paradigms before moving into advanced electives, capstone work, and professional practice. This aligns with the proposal’s goal to equip graduates with durable foundations rather than narrow tool-based expertise.
