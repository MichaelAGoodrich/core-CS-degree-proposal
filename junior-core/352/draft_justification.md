# CS 352 Draft Justification

**Important note:** CS 352 is **not a new course**. It is a **renumbering of CS 452** to align with the proposed junior core sequence. See [CS452_Topics_and_Schedule.md](CS452_Topics_and_Schedule.md) for the current topics, learning outcomes, and schedule.

## Purpose in the Junior Core

CS 352 anchors the **data and information abstractions** pillar in the junior core. It provides students with the conceptual and practical foundations of modern data systems, from relational modeling and SQL to distributed, large-scale, and AI-integrated data workflows. This course is essential for preparing students to reason about data at scale, design resilient systems, and make informed tradeoffs across storage, consistency, and performance.

## Justification Based on Current CS 452 Content

The existing CS 452 curriculum already covers the core competencies expected of a modern database and data systems course. Renumbering the course to CS 352 ensures these competencies are positioned appropriately in the junior core alongside algorithms, programming languages, software engineering, and AI.

Key coverage includes:

- **Relational Foundations (SQL + Modeling):**
	Students gain mastery in SQL querying and develop proficiency in data modeling, schema design, and table relationships—skills essential for any computing professional.

- **AI-Enhanced Data Workflows:**
	The course includes agentic AI, RAG, and semantic search, exposing students to modern data retrieval paradigms that bridge databases and AI systems.

- **NoSQL and Distributed Data Systems:**
	Topics include consistent hashing, replication, resilience, and availability, helping students understand system design tradeoffs in high-velocity data environments.

- **Large-Scale Data Systems:**
	Introduces Spark, Kafka, Kusto, data warehousing, and ETL, which reflect contemporary industry practice for large data processing.

- **Security and Reliability:**
	Students study transaction recovery, indexes, storage, and data security (including row-level security), emphasizing responsible system design.


## Learning Outcomes Alignment

By the end of CS 352, students will be able to:

1. Design normalized relational schemas and implement complex SQL queries.
2. Model data for real-world systems and justify design decisions.
3. Evaluate and apply NoSQL and distributed data architectures.
4. Explain the role of data pipelines, warehousing, and large-scale processing systems.
5. Integrate AI-driven retrieval and semantic search within data systems.
6. Apply security, reliability, and performance principles to database design.
7. Communicate and defend design choices in interview-style evaluations.

## Why It Belongs in the Junior Core

Database systems are foundational to nearly every modern computing domain. Placing this course in the junior core ensures all CS graduates, regardless of later specialization, can reason about data as a first-class abstraction. This aligns directly with the proposed degree’s emphasis on enduring abstractions and prepares students for advanced electives, capstone work, and industry expectations.
