# CS 324 Draft Justification

**Status note:** CS 324 already exists, and **no changes are required at this time**. This document explains why it belongs in the junior core.

## Purpose in the Junior Core

CS 324 anchors the **systems and resource abstractions** pillar in the junior core. It ensures every student understands how software interacts with the operating system, hardware resources, and networked environments—knowledge that underpins performance, reliability, and scalability in modern systems.

## Justification Based on Current Course Overview

The existing CS 324 description includes core systems topics that are foundational for the broader curriculum:

- **Linux systems programming** as the practical basis for low-level system interaction
- **Multiprocessing and concurrency** for performance and correctness
- **Exceptional control flow** for robust systems
- **Caching and memory behavior** for efficiency
- **Sockets, protocols, and non-blocking I/O** for networked and scalable systems

These topics directly support later coursework in databases, AI systems, and software engineering by grounding students in how real systems manage resources and communication.

## Why It Belongs in the Junior Core

Systems programming is a durable abstraction layer that every CS graduate must understand to reason about performance, reliability, and scalability. Including CS 324 in the junior core ensures students have a shared systems foundation before moving into specialized electives and capstone experiences.

## Future Consideration for AI-Across-the-Curriculum

As the curriculum integrates AI more broadly, CS 324 may eventually consider **computing for AI systems**, such as GPU-based computation and accelerator-aware programming models. This would strengthen alignment with AI-intensive workloads while preserving the course’s core systems focus.
